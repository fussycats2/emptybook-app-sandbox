# 디자인·코드 점검 노트

> 2026-04-24 · 전체 UI 다듬기 + 코드 품질 점검 세션

20개 라우트와 공용 컴포넌트를 훑으면서 어색한 부분과 버그를 함께 정리한 기록.
작업 범위는 UI 폴리싱과 클라이언트 코드 버그 픽스까지이며, Auth/Realtime/결제 PG
등 서버 로직 연동은 이번 작업에 **포함되지 않음** (`CLAUDE.md` 미완성 표 참고).

## 1. 디자인 / UX 수정

### 피드 카드 (`components/ui/BookCard.tsx`)
- 위치(`loc`)가 없고 시간(`date`)만 있을 때 `· 날짜` 꼴로 렌더링되던 구분자
  버그 수정. 양쪽이 모두 존재할 때만 `·`를 그리도록 조건 분리.

### 도서 상세 (`app/books/[id]/page.tsx`)
- 판매자 행의 `LocationOnRoundedIcon`을 `Typography`에 `verticalAlign: -2`로
  박아서 세로 정렬이 어긋나던 걸 `Stack direction="row"` 인라인으로 교체.
- 상단 스크롤 헤더 제목: `color: transparent → ink` 토글 방식에서 `opacity`
  페이드로 바꾸고 말줄임(`white-space: nowrap; overflow: hidden; text-overflow:
  ellipsis`) 처리.
- 조회수 계산 `(likes * 5 + 32)`만 단독으로 떠 있던 부분을
  **조회 / 찜 / 채팅** 3종 카운터로 재구성해 아이콘·라벨 일관화.

### 등록 페이지 (`app/register/page.tsx`)
- 상단 타이틀 "사진 N/10"과 추가 버튼에 같은 카운트가 중복 표기되던 걸
  버튼은 **"사진 추가"** 레이블로 변경.
- 10장 초과 시 토스트 + 버튼 opacity로 상한 안내.
- "상세 설명"에 후기 페이지와 동일한 `/500` 글자수 카운터 추가, `.slice(0, 500)`
  으로 상한 강제.

### 결제 / 결제 완료 (`app/checkout/[id]`)
- 무료나눔 거래에서도 "결제 수단", "쿠폰/포인트" 섹션이 노출되던 걸 `!book.free`
  조건부로 숨김.
- 하단 금액 박스는 무료나눔이면 안내 문구로 교체 (상품금액/배송비/쿠폰할인/총액
  대신).
- 결제 완료 페이지의 주문 정보 테이블에서 "결제수단: 카카오페이" 행을 무료나눔일
  때는 숨기고 "결제금액 → 나눔 금액" 으로 라벨 변경.

### 홈 피드 (`app/home/page.tsx` + `components/ui/MannerTemperature.tsx`)
- "인기 판매자" 카드 폭 140 → 160px, 이름 말줄임(`textOverflow: ellipsis`) 추가로
  매너온도 레일이 눌리던 문제 해결.
- `MannerTemperature`의 `sm` 최소 폭 96 → 88로 조정해 좁은 부모에서도 호환.

### LocationChip (`components/ui/LocationChip.tsx`)
- 장식용으로만 있던 드롭다운 화살표에 호버 배경(`palette.lineSoft`)과 타깃
  영역(`padding + borderRadius 999`) 추가.
- 홈 상단에서 `onClick`으로 "동네 변경은 준비 중이에요" 토스트 연결해 기능
  부재를 드러냄.

### 설정 (`app/mypage/settings/page.tsx`)
- `Row` 컴포넌트가 `onClick` 여부와 관계없이 `>` 화살표와 pointer 커서를 보여
  줘서 비인터랙티브 항목(예: "앱 버전")도 클릭 가능한 것처럼 보이던 걸 보정.
  `interactive` 분기로 화살표·커서·호버 모두 제거.
- "앱 버전"을 정보성(`info: true`) 항목으로 표시하고 토스트 바인딩 해제.

### Sticky / z-index 정리
- `AppHeader`, 홈/검색/주문내역 상단 바, `FixedFooter`에 들어가 있던
  `position: sticky`가 **실제로 동작하지 않던 상태** (플렉스 컬럼에서 스크롤
  조상이 아닌 자기 자신에 걸려 있었음). sticky는 전부 제거하고 `z-index`만
  낮춰서(BottomSheet 등 오버레이와 충돌 예방) 유지.

### 상태 뱃지 (`components/ui/StatusBadge.tsx`)
- `SaleStatus`에 `canceled` 케이스 추가 (회갈색 톤: bg `#F5EEEE`, fg `#9A7A7A`).
  `app/mypage/orders/page.tsx`의 `STATUS_TO_BADGE["취소"]`가 `sold`로 매핑돼
  "거래완료" 뱃지로 보이던 버그를 `canceled`로 교정.

## 2. 코드 품질 / 버그 픽스

### 채팅 상세 데이터 연결 (`app/chat/[id]/page.tsx`)
- `fetchBook(params.id)`에 채팅방 id(`c-1` 등)를 넘겨서 **책이 영영 로드되지
  않던 버그**. `fetchChat(params.id) → chat.bookId → fetchBook(bookId)` 순서로
  수정. 헤더의 판매자 이름·지역도 하드코딩 대신 `chat.user` / `book.region` 사용.

### 거래 확정 페이지 (`app/orders/[id]/page.tsx`)
- "채식주의자 / 책방마니아 / 2024.01.16" 하드코딩 제거.
- `fetchOrder(params.id) → fetchBook(order.bookId)` 로 실제 주문·책 데이터에서
  제목·판매자·구매일 표시.

### 검색 필터 실제 반영 (`app/search/page.tsx`)
- `filter.states / priceRange / freeOnly`가 결과에 전혀 반영되지 않던 걸
  `filtered` 레이어 추가해 클라이언트에서 필터링.
  - `freeOnly`: `!b.free` 제외.
  - `states`: 라벨("최상/상/중/하", "A+급/A급/..")을 `BookState` 키로 역매핑.
  - `priceRange`: 무료 아닌 상품만 `[min, max]` 구간 필터.
- `isResultMode` 판정도 `q + category` 에서 `freeOnly`, `states`까지 포함하도록
  확장.

### 타입 정리
- `app/mypage/orders/page.tsx`: `orders: any[] | null` → `OrderRow[] | null`,
  `STATUS_TO_BADGE`도 `Record<OrderRow["status"], SaleStatus>` 로 좁힘.
- `app/chat/page.tsx`: `chats: any[] | null` → `ChatRow[] | null`, 불필요한
  `as SaleStatus` 캐스트 제거.

## 4. 이번 작업 범위 밖 (여전히 미연동)

이번 폴리싱에서는 다루지 않음 — `CLAUDE.md` "미완성" 표에 이미 정리되어 있음.

- Supabase Auth 연동 및 `middleware.ts` 인증 가드
- 채팅 Realtime 구독 훅 (`fetchChat` 은 여전히 mock 고정)
- 이미지 Storage 업로드, ISBN 외부 API, 결제 PG, 알림 Edge Functions
- `toggleLike(bookId)` 등 찜 토글 API
- 후기 저장 (`createReview`) / 프로필 수정 저장 로직

이 항목들은 서버 연동이 필요하므로 UI 폴리싱과는 별도 작업으로 묶는 편이
안전함.
