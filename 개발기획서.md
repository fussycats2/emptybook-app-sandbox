## 3. 엔티티 관계 (Entity Relationship)

| 엔티티          | 관계 | 대상 엔티티     | 설명                                           |
| :-------------- | :--: | :-------------- | :--------------------------------------------- |
| **User**        | 1:N  | **Book**        | 한 유저는 여러 도서를 등록(판매)할 수 있음     |
| **Book**        | 1:1  | **Transaction** | 등록된 도서는 하나의 최종 거래로 연결됨        |
| **User**        | 1:N  | **ChatRoom**    | 한 유저는 여러 거래 상대와 채팅방을 생성함     |
| **Transaction** | 1:1  | **Review**      | 거래가 완료된 건에 대해 하나의 후기가 작성됨   |
| **Transaction** | 1:1  | **Payment**     | 거래 시 선택한 결제 수단 및 금액 정보가 귀속됨 |

## 4. 상세 개발 스펙 (Technical Specs)

### **A. Backend & Database (Supabase)**

- **Auth**: Email 및 소셜 로그인(OAuth) 연동, `profiles` 테이블과 자동 트리거 연결.
- **Database**: PostgreSQL 활용.
  - `books`: 제목, 저자, 출판사, ISBN, 상태, 가격, 판매자ID, 위치.
  - `transactions`: 구매자ID, 도서ID, 상태(판매중/배송중/완료), 거래방식.
- **Realtime**: 채팅 기능 구현을 위해 `messages` 테이블의 Insert 이벤트를 실시간 구독.
- **Storage**: 도서 실물 사진 업로드를 위한 `book-images` 버킷 운영 (WebP 변환 권장).

### **B. Frontend (Next.js & MUI)**

- **App Router**: SSR(Server Side Rendering)을 통한 SEO 최적화(도서 상세 페이지 등).
- **MUI Components**:
  - **Cards/Accordions**: 도서 리스트 및 마이페이지 내역 표현.
  - **Bottom Navigation**: 모바일 웹 최적화를 위한 5탭 구조.
  - **Modals**: 검색 필터 및 결제 수단 선택 시 사용.
- **Form Management**: 도서 등록 시 정규식 검증(가격, 연락처 등) 및 상태 관리.

### **C. Interface & API**

- **External API**: 네이버/카카오 도서 검색 API 연동을 통한 ISBN 기반 데이터 수집.
- **Payment Gateway**: 가상 결제 모듈(Mock) 또는 외부 PG사 UI 연동.
- **Notification**: Supabase Edge Functions를 활용한 상태 변경(배송 시작 등) 알림 푸시.

## 5. 단계별 개발 로드맵

1.  **1단계**: 서비스 기본 골격(MUI 테마 설정) 및 Supabase Auth 연동.
2.  **2단계**: 도서 등록(ISBN API 연동) 및 메인 리스트(DB Fetch) 구현.
3.  **3단계**: 채팅 UI 및 실시간 메시징 로직 구현.
4.  **4단계**: 결제 프로세스 모의 연동 및 거래 상태 머신(State Machine) 완성.
5.  **5단계**: 후기 시스템 및 마이페이지 활동 로그 정교화.
